python object

彻底搞清楚python对象

python对象是什么

在python界有一句话: "python一切都是对象"。

整数，字符串，dict, set, 函数，模块，文件等都是对象。当然也包括使用class关键字自定义的对象

我们知道，在面向对象程序设计中, “对象”是一个实体，类是实体的抽象特点的定义，包括定义数据的形式和数据的操作。类的实例后得到一个对象。

对于python对象而言，有三个问题绕不过去:

  1. 什么是实例化python对象的类
  2. python类定义了哪些数据以及数据的操作
  3. 怎么去”实例化“这个类

搞清楚这三个问题，才能理解“python”一切是对象这个至理名言。

先说上面三个问题的答案:

  1. python的“类”就是C语言中定义的结构体
  2. 结构体中定义了类的数据和操作
  3. ”实例化“C语言的结构体即可得到python对象

现在，你可能迫切想知道，python中类的结构体得到"长"什么样。 那我们首先看一下这个结构体的定义：

typedef struct _typeobject {
    PyObject_VAR_HEAD   /* 公共头 */
    const char *tp_name; /* 打印 "<module>.<name>" */
    Py_ssize_t tp_basicsize, tp_itemsize; /* 定义对象的大小和对象的个数，用于分配内存大小 */

    /* 需要实现的标准操作 */
    destructor tp_dealloc;
    printfunc tp_print;  // 定义了打印对象的方法
    getattrfunc tp_getattr;
    setattrfunc tp_setattr;
    PyAsyncMethods *tp_as_async; /* formerly known as tp_compare (Python 2)
                                    or tp_reserved (Python 3) */
    reprfunc tp_repr;

    /* Method suites for standard classes */

    PyNumberMethods *tp_as_number;  // 定义类似于数学操作集合
    PySequenceMethods *tp_as_sequence; // 定义类似于list的操作集合
    PyMappingMethods *tp_as_mapping; // 定义了类似于map操作集合

    /* More standard operations (here for binary compatibility) */

    hashfunc tp_hash; // 定义了对象的hash操作
    ternaryfunc tp_call; // 定义call(对象)操作
    reprfunc tp_str; // 定义了str(对象)的操作
    getattrofunc tp_getattro; // 定义了getattr的操作
    setattrofunc tp_setattro; // 定义了setattr的操作

    /* Functions to access object as input/output buffer */
    PyBufferProcs *tp_as_buffer;

    /* Flags to define presence of optional/expanded features */
    unsigned long tp_flags;

    const char *tp_doc; /* Documentation string */

    /* call function for all accessible objects */
    traverseproc tp_traverse;

    /* delete references to contained objects */
    inquiry tp_clear;

    /* rich comparisons */
    richcmpfunc tp_richcompare;

    /* weak reference enabler */
    Py_ssize_t tp_weaklistoffset;

    /* Iterators */
    getiterfunc tp_iter;
    iternextfunc tp_iternext;

    /* Attribute descriptor and subclassing stuff */
    struct PyMethodDef *tp_methods;
    struct PyMemberDef *tp_members;
    struct PyGetSetDef *tp_getset;
    struct _typeobject *tp_base;
    PyObject *tp_dict;
    descrgetfunc tp_descr_get;
    descrsetfunc tp_descr_set;
    Py_ssize_t tp_dictoffset;
    initproc tp_init;  // 定义了__init__操作
    allocfunc tp_alloc;
    newfunc tp_new; //  定义了__new__操作
    freefunc tp_free; /* Low-level free-memory routine */
    inquiry tp_is_gc; /* For PyObject_IS_GC */
    PyObject *tp_bases; // 保存所有的父类
    PyObject *tp_mro;  // 定义了查找父类的规则
    PyObject *tp_cache;
    PyObject *tp_subclasses;
    PyObject *tp_weaklist;
    destructor tp_del;

    /* Type attribute cache version tag. Added in version 2.6 */
    unsigned int tp_version_tag;

    destructor tp_finalize;

} PyTypeObject;

初始化 PyTypeObject

一步一步来，我们先需要知道PyObject_VAR_HEAD 头的结构体

# PyObject_VAR_HEAD是一个宏定义
#define PyObject_VAR_HEAD               \
    PyObject_HEAD                       \
    Py_ssize_t ob_size;  # 多了一个ob_size

# define PyObject_HEAD \
    int ob_refcnt; \ # 和引用计算有关
    struct _typeobject *ob_type; # 敲黑板_typeobject就是PyTypeObject结构体标签，也就是说ob_type这个指针必须指向PyTypeObject

这里涉及到一个概念，定长对象和非定长对象，它们对应的头分别是 PyObject_HEAD， PyObject_VAR_HEAD, 所谓定长就是长度是固定的，例如整数，非定长是长度不固定，例如字符串，python还有一个概念是可变和不可变对象，不要搞混淆了。

对于PyObject_VAR_HEAD来说 初始化这个头部需要三个信息:

ob_refcnt # 引用计算
ob_type # 指向一个PyTypeObject的指针
ob_size # 对象所保存的内容的个数，比如字符串"python"那么ob_size 就是6

ob_refcnt和ob_size都好说，重点是ob_type怎么初始化。

请看下面的例子

In [14]: class A(object): pass
In [15]: a = A()

从面向对象的角度来讲，a = A()执行后，a是A类的实例化对象，可以说 a is_instance_of A， 使用python函数表示：isinstance(a, A) = true

又可以描述为：A类实例化后得到a， python中怎么描述“A实例化得到a”这个关系呢，答案是ob_type

如果b对象的ob_type是B, 那么我们可以说“B实例化后得到b”

使用type函数或者__class__属性可以获取到一个对象的instance类

In [16]: type(a)
Out[16]: __main__.A

In [17]: a.__class__
Out[17]: __main__.A  # 这里的输出就是tp_name的赋值，记得哦

所以 对象的ob_type属性的赋值决定了该对象的is_instance_of的对象。

比如我们会说，a对象实例化后得到b对象，实际上就是b.ob_type = a

在python中，我们可以说，b的类型是a。

这里有推导出python另外一个非常不同于其他语言的描述： 实例化一个A对象可以得到B对象

其实说白了了，就是B.ob_type = A , 所以，python的中“实例化”这个名词和面向对象的实例化不太一样！！！

理解这一点至关重要！！！！！

除了PyObject_VAR_HEAD初始化以外，还需要关注的是tp_name,tp_basicsize, tp_itemsize, tp_basicsize和tp_itemsize 决定了分配内存的大小，接下来就是一些方法的初始化, 这里很神奇的我们常用对象的方法就是在这里初始化,例如 print, new, init, 字符串操作集合，list操作集合等

到目前为止，我们学会了怎么初始化一个PyTypeObject，我们现在才是初始化第一个对象 PyType_Type, 这就是大名鼎鼎的type对象 (PyType_Type就是type，下面文字描述统一为type对象)

PyTypeObject PyType_Type = {
    PyVarObject_HEAD_INIT(&PyType_Type, 0) 的父类也是type
    "type", // 初始化tp_name
    sizeof(PyHeadTypeObject), // 初始化tp_basicsize
    sizeof(PyMemBerDef), // 初始化tp_itemsize
    ...省略其他函数和属性的初始化...
}

这里需要 特别关注的是PyType_Type 的 ob_type赋值指向的是自己，说明了PyType_Type是由PyType_Type类实例化得到的，可以理解为'自己初始化了自己'

# type代表的就是PyType_Type对象
In [18]: type.__class__
Out[18]: type  # type的实例化对象还是自己

说了这么半天，我们终于搞清楚了type对象的生成过程，那type对象有什么用, 看下文之前，我们在建立几个概念:

1. PyType_Type对象是通过PyTypeObject 结构体"初始化"得到的

2. PyType_Type是所有类型对象>的祖宗，即所有类型对象都是通过PyType_Type实例化得到的（知道“实例化”怎么理解了吗）

所以，PyType_Type对象的用处就是：生成“类型对象”，类型对象包括PyInt_Type, PyString_Type, PyDict_Type等内置对象。

通过类型对象的实例化可以得到一个具体的对象。

这句话后面还会详细解释，我们先看怎么通过PyType_Type实例化得到类型对象

以PyInt_Type的生成过程为例：

PyTypeObject PyInt_Type = {
PyVarObject_HEAD_INIT(&PyType_Type, 0) // 初始化头部，注意啦，这里用到了上一步生成的PyType_对象
    "int", // 初始化tp_name
    sizeof(PyIntObject), // 初始化tp_basicsize，这里用到了PyIntObject这个结构体，我们接下来会聊到
    0 // 初始化tp_itemsize，因为int对象是固定长度对象，没有item属性，所以是0

    ...省略其他函数和属性的初始化...
}

根据上面的代码我们可以得到两个重要的信息：

  1. PyInt_Type对象是初始化PyTypeObject对象得到的
  2. 初始化中ob_type取值为PyType_Type，换句话说，实例化PyType_Type对象得到了PyInt_Type
In [20]: isinstance(int, type)
Out[20]: True

得到PyInt_Type对象后，我们就可以通过这个对象实例化出来一个整数

实例化PyIntObject

在了解怎么创建一个python整数对象之前，我们在回顾前文的一个概念:

  1. 所有的对象都是C语言中的结构体
  2. A对象通过实例化可以得到B对象

于是，我们需要知道

  1. 整数对象的结构体是什么样
  2. 整数对象是通过什么对象实例化得到的，即整数对象的ob_type是什么
typedef struct {
    PyObject_HEAD # PyObject_HEAD是一个宏定义
    long ob_ival; # 整数的值
} PyIntObject  # PyIntObject结构体就是整数对象的结构体

整数对象的ob_type 是PyInt_Type. 下面是生成一个整数对象函数

使用函数`PyObject *PyInt_FromLong(long ival)` 既可创建int对象

PyInt_FromLong(long ival)
{
    ...省略...

    v = free_list;
    free_list = (PyIntObject *)Py_TYPE(v);

    ******这里使用到了PyInt_type, 值为ival *******
    PyObject_INIT(v, &PyInt_Type);
    v->ob_ival = ival;

    ...省略...

    return (PyObject *) v;
}

到这里，我们基本可以总结:

  1. 初始化PyTypeObject结构体，ob_type=PyType_Type可以得到PyType_Type对象
  2. 初始化PyTypeObject结构体，ob_type=PyType_Type 实例化后可以得到PyInt_Type
  3. 初始化PyIntObject结构体，ob_type=PyInt_Type可以得到整数对象 4 intObject对象的类型是PyInt_Type,PyInt_Type对象的类型是PyType_Type, PyType_Type对象的类型是PyType_Type

到最后，我们可以负责的说，初始化一个对象分为两步:

  1. 给我一个对象的结构体，
  2. 指定这个对象的ob_type

还有一个有趣的概念是：a.ob_type = A， 那么A对象可以称为a对象的元类(metaClass)。

说白了，理解了ob_type，相当于理解了整个python对象创建的逻辑。

到这一步，我们就完成了int对象的创建。可以使用type函数验证一下:

>>> type(int)
<type 'type'>  # 说明PyInt_Type 是PyType_Type实例化得到的

>>> a = int(1)
>>> type(a)
<type 'int'> # 说明PyIntObject对象是PyInt_Type对象实例化得到的

>>> type(type)
<type 'type'> # 说明type对象是PyType_Type 实例化得到的

初始化PyStringObject对象

PyStringObject的ob_type对象是PyString_Type, 也就是说PyStringObject是PyString_Type实例化得到的

PyTypeObject PyString_Type = {
    PyVarObject_HEAD_INIT(&PyType_Type, 0) // 初始化头
    "str",  // tp_name
    PyStringObject_SIZE, // tp_basicsize
    sizeof(char), // tp_itemsize
    string_dealloc,
    (printfunc)string_print,
    0,
    0,
    0,
     string_repr,
    &string_as_number,   // 初始化了数字操作集合
    &string_as_sequence, // 初始化了list操作集合
    &string_as_mapping, // 初始化了map操作集合
    (hashfunc)string_hash,

PyStringObject结构如下:

typedef struct {
    PyObject_VAR_HEAD // 头部信息
    long ob_shash; // 是否hash过了
    int ob_sstate; // 优化相关的
    char ob_sval[1]; // 指向维护实际的字符串内容的内存地址
} PyStringObject;

于是 生成一个PyStringObject对象的方法如下：

PyString_FromStringAndSize (const char *str, Py_ssize_t size)
{
    .....
    op = (PyStringObject *)PyObject_MALLOC(PyStringObject_SIZE + size); // 初始化了PyStringObject
    PyObject_INIT_VAR(op, &PyString_Type, size); // 使用了PyString_Type，注意ob_size的赋值
    op->ob_shash = -1;
    op->ob_sstate = SSTATE_NOT_INTERNED;
    .....
}

类似的，python中dict,set, map等结构都是类似于这种方式创建的

Comon Header

对比PyTypeObject, PyIntObject, PyStringObject， 我们发现，他们有一个公共的头，那就是PyObject_HEAD

其实不止这三个结构体，python里面所有的对象都有PyObject_HEAD这个头文件.

这意味着，所有对象这样都有一个公共的结构体

typedef {
    Python_HEAD # 通过的head
} PyObject

于是我们可以看到python源码中，很多函数返回的都是(Pyobject *p) 指针

指针函数

在PyTypeObject结构体中，我们可以看到很多函数需要初始化，这些函数初始化之后，就可以操作结构体中的“数据”，PyTypeObject的函数大概分为这几类:

  1. 初始化操作 [tp_new, tp_init]
  2. 类型化操作函数[tp_as_number, tp_as_sequence，tp_as_mapping]
  3. 常规操作函数[tp_hash, tp_call, tp_compare, tp_iter, tp_iternext, tp_print]
  4. Python Attribute descriptor [tp_dict,tp_descr_get, tp_descr_set ]
  5. 内存管理函数[tp_allocs, tp_frees, tp_maxalloc]

在python内置类型中，int类型初始化tp_as_number，string类型初始化了tp_as_number， tp_as_sequence，tp_as_mapping

所有，猛然惊醒，如果我们想要int对象也具有list对象的某些行为，只需要实现tp_as_sequence就可以了

这里有引出另外一个话题，怎么去自定义python的对象, 这个是我们接下来需要说明的
